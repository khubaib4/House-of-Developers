Fix the canonical URL trailing slash issue across all pages.

PROBLEM IDENTIFIED:
The canonical URLs currently have inconsistent trailing slashes. 
Some pages have trailing slashes in canonical tags while the actual 
URLs don't, or vice versa. This causes "Canonicalised" warnings in 
SEO tools and can hurt search rankings.

SOLUTION NEEDED:
Standardize all canonical URLs to NOT use trailing slashes (except 
for the homepage if it must).

EXAMPLE OF CORRECT IMPLEMENTATION:
```typescript
import { Helmet } from 'react-helmet-async';
import { useLocation } from 'wouter';

export function SEO({ 
  title, 
  description, 
  image, 
  type = 'website' 
}) {
  const [location] = useLocation();
  
  // Remove trailing slash for canonical URL
  const pathname = location === '/' ? '' : location.replace(/\/$/, '');
  const canonicalUrl = `https://houseofdevelopers.co.uk${pathname}`;
  
  return (
    <Helmet>
      <title>{title}</title>
      <meta name="description" content={description} />
      
      {/* Canonical URL - no trailing slash */}
      <link rel="canonical" href={canonicalUrl} />
      
      {/* Open Graph */}
      <meta property="og:url" content={canonicalUrl} />
      <meta property="og:title" content={title} />
      <meta property="og:description" content={description} />
      <meta property="og:type" content={type} />
      {image && <meta property="og:image" content={image} />}
      
      {/* Twitter Card */}
      <meta name="twitter:card" content="summary_large_image" />
      <meta name="twitter:title" content={title} />
      <meta name="twitter:description" content={description} />
      {image && <meta name="twitter:image" content={image} />}
    </Helmet>
  );
}
```
