COMPONENT 1: CTA SECTION (/components/ui/CTASection.tsx)

Create a compelling CTA section to drive consultation bookings.

PROPS:
- variant: 'default' | 'gradient' | 'bordered' | 'minimal'
- title: string
- description: string
- primaryCTA: { text: string, link: string }
- secondaryCTA?: { text: string, link: string }
- showContactInfo?: boolean (shows phone/email below)
- icon?: LucideIcon (optional decorative icon)
- size?: 'small' | 'medium' | 'large'

DESIGN VARIANTS:

DEFAULT:
- Background: muted
- Padding: py-16 (small), py-20 (medium), py-24 (large)
- Max width: max-w-4xl mx-auto
- Text align: center

GRADIENT:
- Background: linear gradient from primary to accent
- Text color: white
- Box shadow: 2xl with glow
- Premium feel

BORDERED:
- Border: 2px border-primary
- Background: card
- Subtle shadow

MINIMAL:
- No background
- Simple border-t
- Clean and understated

STRUCTURE:
- Icon (if provided, size 48, primary color, mb-4, centered)
- Title (text-3xl md:text-4xl font-bold)
- Description (text-lg text-muted-foreground or white/90, mt-4, max-w-2xl mx-auto)
- Buttons container (mt-8, flex gap-4, justify-center, flex-wrap):
  * Primary button:
    - If gradient variant: bg-white text-primary
    - Else: bg-accent text-white
    - Height: h-12 md:h-14
    - Padding: px-8
    - Rounded: rounded-lg
    - Font: font-semibold text-lg
    - Hover: scale 1.05, shadow
  * Secondary button (if provided):
    - Border variant
    - Same size as primary
    - Hover effects
- Contact info (if showContactInfo):
  * mt-8, flex gap-6, justify-center, text-sm
  * Phone icon + number
  * Mail icon + email
  * Each clickable

RESPONSIVE:
- Mobile: Stack buttons vertically, smaller text
- Desktop: Horizontal buttons

USAGE EXAMPLES:
- Between sections as conversion points
- End of page final CTA
- After feature showcases
- Multiple times per page with different messaging

GLOBAL REQUIREMENTS FOR ALL COMPONENTS:

1. TYPESCRIPT:
   - Proper TypeScript interfaces
   - Type-safe props
   - Default values where needed
   - Export interfaces for reuse

2. RESPONSIVE DESIGN:
   - Mobile-first approach
   - Test: 375px, 768px, 1024px, 1440px
   - Smooth breakpoint transitions
   - Touch-friendly on mobile

3. DARK/LIGHT MODE:
   - Use theme colors consistently
   - Test in both modes
   - Proper contrast ratios
   - Readable in all conditions

4. ANIMATIONS:
   - Framer Motion for complex animations
   - CSS transitions for simple effects
   - Duration: 200-300ms for UI, 500-800ms for page elements
   - Respect prefers-reduced-motion
   - Smooth, not jarring

5. ACCESSIBILITY:
   - Semantic HTML (section, article, etc.)
   - ARIA labels and roles
   - Keyboard navigation (Tab, Enter, Escape)
   - Focus indicators (ring-2 ring-primary)
   - Screen reader friendly
   - Color contrast WCAG AA minimum

6. PERFORMANCE:
   - Lazy load images
   - Optimize re-renders (React.memo where needed)
   - No layout shifts
   - Fast initial paint
   - Efficient animations (transform, opacity)

7. STYLING:
   - Tailwind CSS utilities
   - Consistent spacing scale
   - Match design system
   - Professional polish
   - Hover states on interactive elements

8. DOCUMENTATION:
   - JSDoc comments for props
   - Usage examples in comments
   - Clear prop descriptions
   - Default value documentation

Build all 8 components as production-ready, reusable modules in /components/ui/. Each component should be self-contained, well-typed, and thoroughly tested for responsiveness and accessibility.
