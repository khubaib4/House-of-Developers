Create 8 reusable components for House of Developers CMS service pages. These components will be used across multiple pages for consistency and efficiency.

COMPONENT 1: PRICING CARD (/components/ui/PricingCard.tsx)

Create a professional pricing card component.

PROPS:
- packageName: string (e.g., "Starter", "Professional", "Enterprise")
- price: string (e.g., "From £3,000", "Custom Quote")
- description: string (brief tagline)
- features: array of strings (what's included)
- isPopular: boolean (shows "Popular" badge)
- ctaText: string (default: "Get Started")
- ctaLink: string (where button links)
- billingCycle: string (optional, e.g., "one-time", "monthly")

DESIGN:
- Background: card with border
- Padding: p-8
- Border radius: rounded-xl
- Hover: shadow-lg, border-primary transition
- If isPopular: 
  * Border: 2px border-primary
  * Badge: "Most Popular" (top-right, bg-accent, text-white)
  * Scale slightly larger (scale-105)

STRUCTURE:
- Popular badge (if applicable)
- Package name (text-sm font-semibold text-muted-foreground uppercase tracking-wide)
- Price (text-4xl font-bold, primary color, mt-4)
- Billing cycle (text-sm text-muted-foreground)
- Description (text-muted-foreground, mt-2)
- Divider (border-t, my-6)
- Features list:
  * Each feature: flex items-start gap-2
  * CheckCircle2 icon (primary color, size 16)
  * Feature text (text-sm)
  * Space between: space-y-3
- CTA button (mt-8, full width):
  * If popular: bg-primary (accent color)
  * Else: border button
  * Height: h-12
  * Rounded: rounded-lg
  * Font: font-semibold

RESPONSIVE:
- Mobile: full width
- Desktop: fits in grid

GLOBAL REQUIREMENTS FOR ALL COMPONENTS:

1. TYPESCRIPT:
   - Proper interfaces for all props
   - Type safety
   - Default props where appropriate

2. RESPONSIVE:
   - Mobile-first design
   - Test on 375px, 768px, 1024px
   - Smooth breakpoint transitions

3. DARK/LIGHT MODE:
   - All components work in both modes
   - Use theme colors
   - Proper contrast

4. ANIMATIONS:
   - Framer Motion for entrance animations
   - Smooth transitions (200-300ms)
   - Respect prefers-reduced-motion

5. ACCESSIBILITY:
   - Semantic HTML
   - ARIA labels
   - Keyboard navigation
   - Focus states
   - Screen reader friendly

6. PERFORMANCE:
   - Optimized re-renders
   - Lazy loading where appropriate
   - No layout shifts

7. STYLING:
   - Use Tailwind utilities
   - Consistent spacing
   - Match existing design system
   - Professional and polished

Build all 8 components as separate files in /components/ui/. Make them production-ready, reusable, and well-documented with TypeScript interfaces.
